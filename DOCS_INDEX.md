## Documentation Standards

All documentation follows these standards:

- **Markdown format**: All docs are in Markdown
- **Table of contents**: All major docs include TOC
- **Code examples**: Include practical code examples
- **Cross-references**: Links to related documentation
- **Last updated**: Date stamp at the end of each doc
- **Version**: Version number where applicable

## Contributing to Documentation

When adding or updating documentation:

1. **Update relevant docs**: If adding a feature, update multiple docs if needed
2. **Add examples**: Include code examples where helpful
3. **Cross-reference**: Link to related documentation
4. **Update index**: Update this index if adding new docs
5. **Check formatting**: Ensure consistent formatting
